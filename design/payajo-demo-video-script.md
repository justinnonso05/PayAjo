# PayAjo — Demo Video Script

**Runtime target:** ~4–5 minutes
**Structure:** Web landing page (top → footer) → switch to iOS Simulator → mobile app walkthrough (splash → onboarding → full app)
**Voiceover style:** Confident, plain, no jargon. Short sentences. Let the screen do the work.

---

## 0. Cold Open (0:00–0:15)

**On screen:** Black, then the PayAjo logo fades in on the dark green gradient, centered.

**Voiceover:**
> "Ajo and Esusu have run on trust for generations. We built PayAjo to back that trust with a system — available as a full web app, and a native mobile app for iOS and Android."

**On screen (text overlay, appears under the logo):**
`Web App · iOS App · Android App`

*(This is the one place to explicitly plant "we built both" before anything else — judges should know within the first 15 seconds that this isn't just a mobile demo.)*

## Production Notes

- **Prep a clean test account beforehand** — a fresh signup recorded live looks authentic, but have BVN/OTP test values ready so you're not stuck waiting on real SMS/email during the take.
- **Prep one group with real-looking data already in progress** (a few members, a couple of paid contributions, one pending) for the Group Details / rotation / auto-debit segments — an empty group looks thin on camera.
- **Turn off notification banners / do-not-disturb** on the simulator and browser before recording, so nothing unrelated pops up mid-shot.
- **Record the web section and the mobile section as two separate clips** and edit them together — much easier to re-take one half without re-recording the whole thing.
- **Keep the mouse cursor visible and deliberate** during the web section — slow, obvious hovers on the feature-card wipe and the "How It Works" scroll are what sell those animations; a fast scroll just looks like a blur.
- If a screen doesn't match exactly what's described above by the time you record (copy tweaks, reordered sections), treat this script as the shot list to adapt, not a rigid transcript.
